# OmniSense Lab — 產品需求與願景

## 願景

打造以 **ESP32-C3** 為感測中樞、**Web Bluetooth（PWA）** 為互動介面的物理實驗平台：教學現場可無線取得多通道類比／數位／觸控資料，並以**插件式實驗**擴充（例如 Omni-Forge 等），而**穩定韌體**與**通訊協定**保持長期相容。

## 核心原則

1. **引擎與實驗分離**：`web/core/` 與 `firmware/` 定義協定與資料管線；`experiments/*` 只透過事件與設定 API 取用資料，不直接綁死 BLE 物件。
2. **單一事件匯流排**：感測資料以 `omnisense:data`（相容別名 `omnisense-data`）送出，詳見 `PROTOCOL.md`。
3. **動態載入**：依 `projects.json` 註冊實驗，切換頁面時載入對應 `experiments/<id>/app.js`，離開時 `unmount` 釋放 Canvas／動畫。
4. **韌體慎改**：穩定後以 `firmware/OmniSense_Main` 為凍結基線；教學迭代優先在 Web 完成。

## 非目標（現階段）

- 裝置端 OTA、雲端帳號、多人協作編輯。
- 取代專業數據採集卡之絕對精度與取樣率（以實驗示範為主）。

## 插件範例：Omni-Forge — 弓匠試煉（三階量測）

教學用「鍛弓」敘事與 **R = ρL/A** 物理語彙結合感測：

| 階段 | 名稱 | 意義（感測） |
|------|------|----------------|
| **Void** | 虛空 | 弓弦靜止時之底噪／基準（G2 ADC） |
| **Titan** | 巨力 | 拉滿時之最大形變與魔力能量（ΔADC + 倒數圓圈） |
| **Return** | 回響 | 放箭後回復時間與漂移 |

至少 **5 次**試煉後，**去最高／最低分**再取平均作為最終鑑定分；未達門檻則 UI 要求 **Remake** 方可視為通關。
